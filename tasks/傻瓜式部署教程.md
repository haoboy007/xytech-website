## 第一步：配置 GitHub 免密推送（关键！）

秒哒环境无法交互输入密码，需要配置 Git Token 实现免密推送。

### 1.1 在 GitHub 生成 Token

1. 打开 https://github.com/settings/tokens
2. 点击 **Generate new token (classic)**
3. 填写 Note：`xytech-website-deploy`
4. 勾选权限：
   - ✅ `repo`（完整仓库访问）
   - ✅ `workflow`（如需 GitHub Actions）
5. 点击 **Generate token**
6. **立即复制保存 Token**（页面关闭后不可再见）

### 1.2 在秒哒配置 Git 凭证

在秒哒终端执行以下命令（将 `YOUR_TOKEN` 替换为刚才复制的 Token）：

```bash
cd /workspace/app-bset9i1icp35

# 配置 Git 用户名和邮箱（只用配一次）
git config user.name "haoboy007"
git config user.email "your-email@example.com"

# 配置免密推送 URL（将 YOUR_TOKEN 替换为真实 Token）
git remote set-url origin https://haoboy007:YOUR_TOKEN@github.com/haoboy007/xytech-website.git

# 验证
git remote -v
```

输出应类似：
```
origin  https://haoboy007:ghp_xxxxxxxx@github.com/haoboy007/xytech-website.git (fetch)
origin  https://haoboy007:ghp_xxxxxxxx@github.com/haoboy007/xytech-website.git (push)
```

## 第二步：秒哒端 — 一键构建并推送

### 2.1 创建推送脚本

在秒哒终端执行：

```bash
cat > /workspace/app-bset9i1icp35/push-to-github.sh << 'EOF'
#!/bin/bash
# ============================================
# 秒哒一键构建 + 推送到 GitHub
# ============================================

cd /workspace/app-bset9i1icp35

echo "[1/4] 拉取远程最新代码..."
git pull origin main 2>/dev/null || true

echo "[2/4] 构建生产包..."
npm run build

echo "[3/4] 清理构建产物..."
sed -i '/<link rel="stylesheet" id="fonts-code"/d' dist/index.html

echo "[4/4] 推送到 GitHub..."
git add .
git commit -m "deploy: $(date '+%Y-%m-%d %H:%M:%S')" || true
git push origin main

echo "✅ 推送完成！"
EOF

chmod +x /workspace/app-bset9i1icp35/push-to-github.sh
```

### 2.2 以后每次更新（只需1条命令）

```bash
/workspace/app-bset9i1icp35/push-to-github.sh
```

执行后会自动：
1. 拉取远程代码（防止冲突）
2. 构建项目（`npm run build`）
3. 清理秒哒 CDN 链接
4. 推送到 GitHub

## 第三步：服务器端 — 自动拉取部署

### 3.1 在服务器安装 Git

SSH 登录阿里云服务器，执行：

```bash
yum install -y git
```

### 3.2 克隆仓库到服务器

```bash
mkdir -p /www/wwwroot/xytech-source
cd /www/wwwroot/xytech-source
git clone https://github.com/haoboy007/xytech-website.git .
```

### 3.3 创建自动部署脚本

**重要**：CentOS 7 无法运行 Node.js 20，所以服务器**只拉取 dist 目录**，不构建。

在服务器执行：

```bash
cat > /www/wwwroot/auto-deploy.sh << 'EOF'
#!/bin/bash
# ============================================
# 服务器自动拉取 + 部署脚本
# 由宝塔计划任务每5分钟执行
# ============================================

set -e

SOURCE_DIR="/www/wwwroot/xytech-source"
DEPLOY_DIR="/www/wwwroot/xytech"
LOG_FILE="/www/wwwroot/deploy.log"
LOCK_FILE="/tmp/xytech-deploy.lock"

echo "========================================" >> "$LOG_FILE"
echo "  部署开始 $(date '+%Y-%m-%d %H:%M:%S')" >> "$LOG_FILE"

# 防止并发
if [ -f "$LOCK_FILE" ]; then
    PID=$(cat "$LOCK_FILE" 2>/dev/null || echo "")
    if [ -n "$PID" ] && ps -p "$PID" > /dev/null 2>&1; then
        echo "  [SKIP] 已有部署进程" >> "$LOG_FILE"
        exit 0
    fi
    rm -f "$LOCK_FILE"
fi

echo $$ > "$LOCK_FILE"

cd "$SOURCE_DIR"

# 获取远程最新版本
git fetch origin main
LOCAL=$(git rev-parse HEAD)
REMOTE=$(git rev-parse origin/main 2>/dev/null || echo "$LOCAL")

if [ "$LOCAL" = "$REMOTE" ]; then
    echo "  [OK] 已是最新版本" >> "$LOG_FILE"
    rm -f "$LOCK_FILE"
    exit 0
fi

echo "  [UPDATE] ${LOCAL:0:7} -> ${REMOTE:0:7}" >> "$LOG_FILE"

# 拉取最新代码
git reset --hard origin/main

# 检查 dist 目录是否存在
if [ ! -d "$SOURCE_DIR/dist" ]; then
    echo "  [ERROR] dist目录不存在，需要秒哒端构建后推送" >> "$LOG_FILE"
    rm -f "$LOCK_FILE"
    exit 1
fi

# 部署到网站目录
rm -rf "$DEPLOY_DIR"/*
cp -r "$SOURCE_DIR"/dist/* "$DEPLOY_DIR"/
chown -R www:www "$DEPLOY_DIR" 2>/dev/null || chown -R nginx:nginx "$DEPLOY_DIR" 2>/dev/null || true
chmod -R 755 "$DEPLOY_DIR"

echo "  [DONE] 部署完成 $(git rev-parse --short HEAD)" >> "$LOG_FILE"
rm -f "$LOCK_FILE"
EOF

chmod +x /www/wwwroot/auto-deploy.sh
```

### 3.4 宝塔计划任务配置

1. 登录宝塔面板
2. 点击 **计划任务** → **添加任务**
3. 填写配置：

| 配置项 | 值 |
|--------|-----|
| **任务类型** | Shell脚本 |
| **任务名称** | 雄元官网-自动部署 |
| **执行周期** | 每5分钟 |
| **脚本内容** | `/www/wwwroot/auto-deploy.sh` |

4. 点击 **确定**
5. 点击 **执行** 测试一次
6. 查看日志确认成功

## 第四步：宝塔网站配置

### 4.1 Nginx 配置

宝塔面板 → **网站** → 找到 `www.cnxy.tech` → **设置** → **配置文件**

确保包含以下内容：

```nginx
server {
    listen 80;
    server_name www.cnxy.tech cnxy.tech;
    
    # 网站根目录
    root /www/wwwroot/xytech;
    index index.html;
    
    # Gzip 压缩
    gzip on;
    gzip_types text/plain text/css application/javascript application/json image/svg+xml;
    
    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|svg|woff|woff2|ttf|ico)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # 前端路由支持（关键！）
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

点击 **保存** → **重载配置**

### 4.2 首次手动部署

在服务器执行一次手动部署：

```bash
cd /www/wwwroot/xytech-source
git pull origin main

# 复制 dist 到网站目录
rm -rf /www/wwwroot/xytech/*
cp -r /www/wwwroot/xytech-source/dist/* /www/wwwroot/xytech/
chown -R www:www /www/wwwroot/xytech
chmod -R 755 /www/wwwroot/xytech
```

## 常见问题

### Q1: 秒哒推送报错 "could not read Username"

原因：Git Token 未配置或已过期。

解决：
```bash
cd /workspace/app-bset9i1icp35
git remote set-url origin https://haoboy007:YOUR_NEW_TOKEN@github.com/haoboy007/xytech-website.git
```

### Q2: 服务器拉取报错 "dist 目录不存在"

原因：秒哒推送时构建失败，或 dist 未被提交到 Git。

解决：在秒哒检查构建是否成功：
```bash
cd /workspace/app-bset9i1icp35
ls -la dist/
```

dist 目录必须在 Git 跟踪范围内。

### Q3: 网站白屏或 404

原因：Nginx 缺少前端路由配置。

解决：在 Nginx 配置中添加：
```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

### Q4: 如何查看部署日志

```bash
# 在服务器执行
cat /www/wwwroot/deploy.log
tail -f /www/wwwroot/deploy.log
```
