# 雄元科技官网 - 宝塔面板部署教程

## 一、环境检查

请确认宝塔面板已安装以下软件：
- **Nginx**（建议 1.24+）
- 如果未安装，在宝塔「软件商店」搜索安装

## 二、部署步骤

### 步骤1：创建站点

1. 登录宝塔面板：`https://8.154.33.106:3783/xytech`
2. 左侧菜单 → **网站** → **添加站点**
3. 填写信息：
   - 域名：`www.cnxy.tech`
   - 根目录：`/www/wwwroot/xytech`（默认即可）
   - 数据库：不创建
   - PHP版本：纯静态
4. 点击 **提交**

### 步骤2：上传网站文件

**方式A：通过宝塔文件管理器上传（推荐）**

1. 宝塔左侧 → **文件**
2. 进入目录：`/www/wwwroot/xytech`
3. 点击 **上传** → 选择 `xytech-deploy.zip`
4. 上传完成后，右键点击 zip 文件 → **解压**
5. 解压后，将 `dist` 文件夹内的所有文件 **剪切/移动** 到 `/www/wwwroot/xytech` 根目录
   - 确保 `index.html` 直接在 `/www/wwwroot/xytech/` 下，而不是 `/www/wwwroot/xytech/dist/`

**方式B：通过宝塔终端命令上传**

1. 宝塔左侧 → **终端**
2. 执行以下命令：

```bash
# 进入网站目录
cd /www/wwwroot/xytech

# 下载部署包（请先将 xytech-deploy.zip 上传到宝塔可访问的位置，如 /tmp/）
# 或者通过宝塔文件管理器上传后再解压

# 解压
unzip -o xytech-deploy.zip

# 将dist内容移到根目录
mv dist/* .
mv dist/.* . 2>/dev/null || true
rmdir dist

# 设置权限
chown -R www:www /www/wwwroot/xytech
chmod -R 755 /www/wwwroot/xytech
```

### 步骤3：配置Nginx（关键）

React项目需要配置路由重写，否则刷新页面会404。

1. 宝塔左侧 → **网站** → 找到 `www.cnxy.tech` → **设置**
2. 切换到 **配置文件** 标签
3. 在 `server {` 内添加以下内容（如果已有 location / 则替换）：

```nginx
    location / {
        try_files $uri $uri/ /index.html;
    }
```

完整配置参考（tasks/nginx.conf 文件）：

```nginx
server {
    listen 80;
    server_name www.cnxy.tech cnxy.tech;
    root /www/wwwroot/xytech;
    index index.html;

    # Gzip压缩
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;

    # 缓存静态资源
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # React路由刷新支持（必须）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

4. 点击 **保存**
5. 在「网站」列表中，点击 **重载配置** 或 **重启Nginx**

### 步骤4：配置SSL（HTTPS）

1. 宝塔「网站」→ 找到 `www.cnxy.tech` → **设置**
2. 左侧 → **SSL**
3. 选择 **Let's Encrypt** → 勾选域名 → 点击 **申请**
4. 申请成功后，开启 **强制HTTPS**

### 步骤5：验证部署

1. 浏览器访问：`http://www.cnxy.tech`
2. 检查页面是否正常显示
3. 测试页面刷新功能（如 `/about`、`/contact` 等路由刷新不应404）
4. 检查图片和静态资源是否正常加载

## 三、目录结构验证

部署完成后，`/www/wwwroot/xytech` 目录结构应为：

```
/www/wwwroot/xytech/
├── index.html          ← 首页入口
├── favicon.png
├── favicon.svg
├── icons.svg
├── logo.png
├── partners.jpg
├── assets/
│   ├── index-xxx.js    ← JS打包文件
│   └── index-xxx.css   ← CSS打包文件
└── images/             ← 静态图片
    ├── error/
    ├── logo/
    └── shape/
```

## 四、常见问题

| 问题 | 解决方案 |
|------|----------|
| 页面空白/JS加载失败 | 检查 `index.html` 中 JS/CSS 路径是否为绝对路径 `/assets/...` |
| 路由刷新404 | 确认 Nginx 配置了 `try_files $uri $uri/ /index.html;` |
| 图片不显示 | 检查图片文件是否已上传到正确位置 |
| 权限错误 | 执行 `chown -R www:www /www/wwwroot/xytech` |
| 缓存未更新 | 清除浏览器缓存，或修改文件名强制刷新 |

## 五、更新部署

后续代码更新时，只需：
1. 重新打包生成新的 `xytech-deploy.zip`
2. 上传到服务器覆盖原有文件
3. 重载Nginx配置
