# 雄元科技官网实时同步更新完整方案

> 目标：在秒哒平台修改代码后，雄元科技官网 `https://www.cnxy.tech` 自动同步更新

## 方案一：GitHub Actions 全自动部署（推荐 ★★★★★）

**原理**：秒哒改代码 → `git push` → GitHub 自动构建 → 自动上传到阿里云

### 第一步：初始化 Git 仓库

在秒哒终端执行：

```bash
cd /workspace/app-bset9i1icp35

# 初始化 Git
git init
git add .
git commit -m "v17: 添加合伙人招募页面"

# 在 GitHub 创建新仓库（先在浏览器操作）
# https://github.com/new → 仓库名：xytech-website → 创建

# 关联远程仓库（替换为你的用户名）
git remote add origin https://github.com/你的用户名/xytech-website.git
git branch -M main
git push -u origin main
```

### 第二步：配置 GitHub Secrets（敏感信息）

进入你的 GitHub 仓库 → **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| 名称 | 值 | 说明 |
|------|-----|------|
| `ALIYUN_HOST` | `8.154.33.106` | 阿里云服务器 IP |
| `ALIYUN_PORT` | `22` | SSH 端口（默认22） |
| `ALIYUN_USER` | `root` | SSH 用户名 |
| `ALIYUN_PASSWORD` | 你的密码 | SSH 密码 |
| `ALIYUN_DEPLOY_PATH` | `/www/wwwroot/xytech` | 宝塔网站目录 |

> ⚠️ 如果使用密钥登录，将 `ALIYUN_PASSWORD` 改为 `ALIYUN_SSH_KEY`，值为私钥内容。

### 第三步：GitHub Actions 配置（已创建）

项目已包含 `.github/workflows/deploy.yml`，推送代码后自动触发构建部署。

### 第四步：以后每次更新只需

```bash
cd /workspace/app-bset9i1icp35
git add .
git commit -m "更新内容描述"
git push origin main
```

推送后约 **2-3 分钟**，官网自动更新。可在 GitHub 仓库 → **Actions** 标签查看部署进度。

## 方案二：宝塔面板 Webhook 自动拉取

**原理**：秒哒改代码 → 推送 GitHub → 宝塔 Webhook 触发 → 自动拉取构建

### 第一步：服务器安装 Node.js（宝塔操作）

1. 宝塔面板 → **软件商店** → 搜索 **Node.js 版本管理器** → 安装
2. 安装 Node.js 20 版本

### 第二步：创建项目源码目录

```bash
# 在宝塔终端执行
mkdir -p /www/wwwroot/xytech-source
cd /www/wwwroot/xytech-source
git clone https://github.com/你的用户名/xytech-website.git .
npm install
```

### 第三步：创建自动部署脚本

在服务器创建文件 `/www/wwwroot/deploy.sh`：

```bash
#!/bin/bash
cd /www/wwwroot/xytech-source

# 拉取最新代码
git pull origin main

# 安装依赖
npm install

# 构建
npm run build

# 移除秒哒CDN链接
sed -i '/<link rel="stylesheet" id="fonts-code"/d' dist/index.html

# 部署到网站目录
rm -rf /www/wwwroot/xytech/*
cp -r /www/wwwroot/xytech-source/dist/* /www/wwwroot/xytech/

# 修正目录权限
chown -R www:www /www/wwwroot/xytech

echo "部署完成：$(date)"
```

```bash
chmod +x /www/wwwroot/deploy.sh
```

### 第四步：宝塔计划任务定时执行

宝塔面板 → **计划任务** → 添加任务：

- 任务类型：Shell 脚本
- 任务名称：自动部署雄元官网
- 执行周期：每 5 分钟
- 脚本内容：`/www/wwwroot/deploy.sh`

### 第五步：以后更新

秒哒平台修改后，只需执行：

```bash
git push origin main
```

宝塔每 5 分钟自动拉取最新代码并构建部署。

## 方案三：秒哒一键构建 + 宝塔手动上传（最简单）

**适合**：更新频率不高，或无法配置 GitHub Actions 的场景

### 一键构建脚本（已内置）

秒哒终端执行：

```bash
cd /workspace/app-bset9i1icp35
./deploy.sh
```

这会生成 `/workspace/app-bset9i1icp35/xytech-deploy-final.zip`

### 宝塔手动上传

1. 宝塔面板 → **文件** → 进入 `/www/wwwroot/xytech`
2. 删除旧文件，上传新的 `xytech-deploy-final.zip`
3. 解压到当前目录

## 方案四：秒哒构建 → 宝塔 API 上传（无 SSH）

如果服务器 SSH 端口未开放，可使用宝塔 API 上传文件。

### 第一步：获取宝塔 API 密钥

宝塔面板 → **设置** → **API 接口** → 开启并获取密钥

### 第二步：GitHub Actions 使用宝塔 API

修改 `.github/workflows/deploy.yml`，将 `Deploy via SSH/SCP` 步骤替换为：

```yaml
      - name: Upload to Baota via API
        run: |
          # 创建 zip 包
          cd dist && zip -r ../deploy.zip . && cd ..

          # 上传到宝塔（需替换 BT_PANEL 和 BT_KEY）
          curl -X POST "https://你的宝塔面板地址/files?action=UploadFile" \
            -H "Content-Type: multipart/form-data" \
            -F "file=@deploy.zip" \
            -F "path=/www/wwwroot/xytech" \
            -F "bt_key=${{ secrets.BT_KEY }}"
```

> ⚠️ 此方案需要宝塔面板开启 HTTPS 并开放外部访问，有一定安全风险。

## Nginx 配置（关键！）

无论哪种方案，宝塔面板 Nginx 配置必须添加：

```nginx
location / {
    try_files $uri $uri/ /index.html;
}
```

操作步骤：
1. 宝塔面板 → **网站** → 找到 `www.cnxy.tech` → **设置**
2. **配置文件** 标签
3. 在 `location / { ... }` 内添加 `try_files $uri $uri/ /index.html;`
4. 保存 → 点击 **重载配置**

## 常见问题

### Q1: 推送到 GitHub 后 Actions 没触发？

检查：
- 是否推送到 `main` 或 `master` 分支
- 仓库是否已创建 `.github/workflows/deploy.yml`
- GitHub 仓库 → Actions 标签查看是否有运行记录

### Q2: 部署后页面空白？

检查：
- Nginx 是否配置了 `try_files $uri $uri/ /index.html;`
- `dist` 目录是否正确上传到网站根目录
- 浏览器控制台是否有 JS 报错

### Q3: 部署后样式错乱？

检查：
- 是否移除了秒哒字体 CDN 链接（构建脚本已自动处理）
- CSS 文件是否正确加载

### Q4: 如何回滚到上一版本？

```bash
# 在秒哒终端执行
git log --oneline -5
git revert HEAD   # 回滚最近一条提交
git push origin main
```

GitHub Actions 会自动部署回滚后的版本。
