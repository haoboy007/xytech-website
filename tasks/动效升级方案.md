# 雄元科技官网 Hero 区动效升级方案

## 一、现状诊断：原有粒子动画的问题

| 维度 | 现有方案表现 | 问题分析 |
|------|-----------|---------|
| **视觉秩序** | 60个随机圆点 + 全连接 | 节点分布无规律，连线密集交叉，形成"视觉噪音" |
| **主题契合** | 通用粒子连线 | 与"科技创新/AI/数据"主题无视觉关联 |
| **动线逻辑** | 随机漂移 | 缺乏引导性，没有从左到右或中心发散的视觉叙事 |
| **色彩表现** | 单色紫色（#7B38EE） | 无渐变、无发光、无冷暖对比，显得单薄 |
| **交互体验** | 无响应 | 静态背景，无法与访客产生互动 |
| **性能** | O(n²)距离计算（1770条/帧） | 连线距离计算开销大，缩放性差 |

## 二、新方案：光流拓扑网格（Flowing Topology Mesh）

### 2.1 概念描述

将 Hero 背景从"随机粒子云"升级为**"数据光流拓扑网格"**——以多条水平正弦波为骨架，节点沿波有序运动，形成类似芯片电路、神经网络拓扑的科技感结构。脉冲光点沿路径流动，象征数据在系统中的传输。

**视觉隐喻**：
- 正弦波 = 数据流通道
- 波上节点 = 数据包/计算节点
- 拓扑连线 = 系统架构/网络连接
- 脉冲光点 = 实时数据包传输
- 鼠标涟漪 = 用户介入/交互响应

### 2.2 视觉风格定义

| 元素 | 风格 |
|------|------|
| 背景 | 深邃暗色（#0a0e1a → #121830 径向渐变） |
| 数据流基线 | 低透明度冷灰（rgba(0,229,255,0.06)） |
| 拓扑连线 | 极细线（0.5px），低透明度（0.08-0.15） |
| 脉冲光点 | 青蓝渐变发光（#00E5FF → #7B61FF），shadowBlur 发光 |
| 节点 | 微小圆点（1.5px），冷色系渐变 |
| 鼠标交互区 | 青色涟漪扩散，节点产生场效应位移 |

## 三、新旧方案对比

| 对比维度 | 旧方案（随机粒子） | 新方案（光流拓扑网格） |
|---------|------------------|---------------------|
| **秩序感** | ❌ 无秩序，随机分布 | ✅ 正弦波骨架，网格结构清晰 |
| **主题契合** | ❌ 通用粒子，无业务关联 | ✅ 数据流/神经网络隐喻，契合AI/科技主题 |
| **动线逻辑** | ❌ 无方向，无叙事 | ✅ 水平数据流 + 脉冲传输，有明确视觉动线 |
| **色彩层次** | ❌ 单色，无发光 | ✅ 青蓝霓虹渐变 + glow 效果 |
| **交互性** | ❌ 无交互 | ✅ 鼠标场效应涟漪 |
| **视觉干净度** | ❌ 60粒子×全连接=杂乱 | ✅ 48节点×固定拓扑=清爽 |
| **性能** | ⚠️ O(n²)距离计算 | ✅ O(n)固定连接，开销更低 |
| **辨识度** | ❌ 像无数网站的"标配" | ✅ 独特的科技光流视觉语言 |

## 四、技术实现方案

### 4.1 架构设计

```
Canvas 2D（单一层级，无DOM开销）
├── 背景渐变（fillRect）
├── 6条 FlowLine（正弦波数据流）
│   ├── 基础正弦波路径（stroke，低透明度）
│   ├── 节点（8个/条，fillCircle + glow）
│   ├── 同流线连线（stroke，拓扑感）
│   └── 脉冲光点（gradient fill + shadowBlur，沿路径运动）
├── 跨流线拓扑连线（相邻线之间节点互连）
└── 鼠标交互层
    ├── 涟漪扩散（多层渐变圆环）
    └── 场效应位移（附近节点受鼠标位置影响产生偏移）
```

### 4.2 关键算法

#### 正弦波节点位置计算
```typescript
// 每条流线的节点位置
for (let i = 0; i < nodeCount; i++) {
  const x = leftPadding + i * stepX;
  const phase = time * speed + i * 0.3 + lineIndex * 0.5;
  const y = lineY + Math.sin(phase) * amplitude;
  // 节点位置 = (x, y)
}
```

#### 脉冲光点路径插值
```typescript
// 光点沿流线平滑移动
const t = (time * pulseSpeed) % (nodeCount - 1);
const idx = Math.floor(t);
const frac = t - idx;
// 在节点idx和idx+1之间线性插值
const px = lerp(nodes[idx].x, nodes[idx+1].x, frac);
const py = lerp(nodes[idx].y, nodes[idx+1].y, frac);
```

#### 鼠标场效应（简化的排斥/吸引）
```typescript
// 节点到鼠标的距离
const dx = node.x - mouse.x;
const dy = node.y - mouse.y;
const dist = Math.sqrt(dx*dx + dy*dy);
if (dist < influenceRadius) {
  const force = (1 - dist/influenceRadius) * maxDisplacement;
  node.offsetX = (dx/dist) * force;
  node.offsetY = (dy/dist) * force;
}
```

### 4.3 性能优化策略

| 策略 | 说明 |
|------|------|
| 固定网格结构 | 节点位置通过公式直接计算，无需存储和更新随机状态 |
| 减少节点数 | 从60减至48，减少绘制指令 |
| 固定拓扑连线 | 只连接相邻节点，不计算距离，O(1)判定 |
| 分层透明度 | 基线0.06、连线0.10、节点0.40、脉冲0.90，视觉层次分明 |
| `willReadFrequently: false` | Canvas上下文配置优化 |
| `requestAnimationFrame` 节流 | 后台标签页自动暂停动画 |
| 涟漪对象池 | 复用涟漪对象，避免频繁GC |

### 4.4 技术栈

- **纯 Canvas 2D API** — 零外部依赖，跨浏览器兼容
- **原生 requestAnimationFrame** — 自动同步屏幕刷新率
- **TypeScript** — 类型安全，便于维护
- **CSS backdrop-filter** — Hero区内容层毛玻璃效果（可选增强）

## 五、扩展方向

| 阶段 | 增强项 | 复杂度 |
|------|--------|--------|
