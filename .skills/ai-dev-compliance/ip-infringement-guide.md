# 知识产权侵权审查详细指南

## 一、软件代码侵权

### 1.1 侵权认定标准

软件著作权侵权采用"接触+实质性相似"标准：
- **接触**：被告有机会接触原告软件（如开源发布、公开展示）
- **实质性相似**：两部软件在表达层面（代码结构、组织、序列）存在相似

> ⚠️ 注意：只保护"表达"不保护"思想"。相同的业务逻辑、功能用不同代码实现不构成侵权。

### 1.2 高风险行为

| 行为 | 风险等级 | 说明 |
|------|---------|------|
| 直接复制GitHub私有仓库代码 | 🔴 极高 | 明确侵权 |
| 复制开源项目但未遵守许可证 | 🔴 极高 | 许可证违约+著作权侵权 |
| 复制Stack Overflow代码用于商业产品 | 🟡 中高 | SO内容CC BY-SA 4.0，需署名；小片段通常安全 |
| 使用AI生成的代码与他人代码相似 | 🟡 中 | AI可能"记忆"训练数据中的代码片段，存在争议 |
| 参考他人API设计实现兼容接口 | 🟢 低 | API设计通常不受著作权保护（Google v. Oracle案） |

### 1.3 代码抄袭防护自查

1. **代码指纹比对**：使用工具扫描代码是否与已知开源项目重复
   - 推荐工具：Snyk Code、Black Duck、FOSSology（免费）
2. **AI生成代码审查**：对AI生成的代码进行人工审查，检查是否包含他人版权声明
3. ** clean room 开发**：如已接触他人代码，由不同人员基于功能规格重新实现

## 二、图片与视觉素材侵权

### 2.1 图片侵权重灾区

| 场景 | 风险 | 解决方案 |
|------|------|---------|
| 百度/Google图片直接下载使用 | 🔴 极高 | 改用Unsplash/Pexels/CC0图库 |
| 社交媒体（微博/小红书/Instagram）截图 | 🔴 极高 | 需获得原作者授权 |
| 设计师模板未购买商业授权 | 🟡 高 | 确认购买的是商业许可而非个人许可 |
| AI生成图片用于商标/专利 | 🟡 中高 | 美国USPTO已拒绝纯AI生成图像的商标/专利申请 |
| 用户上传内容（UGC） | 🟡 中 | 需有侵权投诉处理机制和用户赔偿条款 |

### 2.2 免费商用图库推荐

- **Unsplash** (unsplash.com)：CC0协议，可自由商用
- **Pexels** (pexels.com)：Pexels License，免费商用
- **Pixabay** (pixabay.com)：Pixabay License，免费商用
- **阿里云/腾讯云免费素材库**：面向中国开发者的中文素材

### 2.3 字体侵权

字体侵权是中文开发者最容易忽视的风险：

| 字体 | 许可证 | 可商用？ |
|------|--------|---------|
| 微软雅黑 | 微软系统字体 | ❌ 仅系统显示，嵌入分发需授权 |
| 思源黑体/宋体 | Adobe/Google OFL | ✅ 免费商用 |
| 阿里巴巴普惠体 | 阿里免费商用 | ✅ 免费商用 |
| 站酷系列字体 | 站酷免费商用 | ✅ 免费商用 |
| 方正/汉仪字体 | 商业字体 | ❌ 需购买授权 |
| Google Fonts全部 | 各字体OFL/Apache | ✅ 全可商用 |

> 💡 **建议**：Web项目统一使用Google Fonts或阿里巴巴字体；App项目优先使用系统默认字体或确认免费商用字体。

## 三、商标侵权

### 3.1 商标冲突自查步骤

1. **中国**：国家知识产权局商标局官网（sbj.cnipa.gov.cn）商标查询
2. **美国**：USPTO TESS系统
3. **欧盟**：EUIPO eSearch
4. **国际**：WIPO Global Brand Database

### 3.2 高风险命名场景

- 应用名称包含知名品牌词（如"微信助手"、"淘宝精选"）
- Logo颜色/图形与知名品牌高度相似
- 域名/appid含有他人商标

### 3.3 应对策略

- 应用命名前做商标查询
- 避免使用行业通用词汇作为商标（难以注册且易冲突）
- 如收到商标侵权通知，立即评估是否构成合理使用/描述性使用

## 四、专利风险

### 4.1 软件专利现状

- **美国**：软件专利活跃，但 Alice Corp案后抽象概念专利被限缩
- **中国**：软件专利可获得授权，但侵权诉讼中常被主张无效
- **欧盟**：软件本身不可专利，但技术效果的软件实现可专利

### 4.2 一人开发者专利风险实际评估

对于小型应用，专利风险相对较低（目标小、难以发现、诉讼成本不经济），但以下场景需警惕：
- 功能直接对标知名产品核心功能
- 使用他人专利描述的特定技术方案
- 涉及音视频编解码、加密算法等技术领域

### 4.3 低成本FTO（自由实施）分析

1. 在Google Patents搜索核心功能关键词
2. 查看行业龙头企业的专利布局
3. 避免复制专利权利要求的精确技术方案

## 五、AI生成内容的版权

### 5.1 各国/地区立场

| 地区 | 立场 | 参考案例 |
|------|------|---------|
| 中国大陆 | AI生成内容若体现人类独创性安排，可认定为作品 | 菲林律所诉百度案（2018）：AI报告不构成作品 |
| 美国 | 纯AI生成内容不受版权保护，需人类作者身份 | USCO Thaler案：拒绝AI生成图像版权登记 |
| 欧盟 | 类似美国，要求人类智力创作 | 暂无AI版权判例 |
| 英国 | 《版权法》第9条：计算机生成作品版权属于安排创作的人 | 相对宽松 |

### 5.2 实务建议

- 将AI作为"辅助工具"而非"独立创作者"，确保人类在创意选择、编排、修改中发挥实质作用
- 保留人类创作过程的记录（prompt迭代、人工编辑痕迹）
- 在AI生成内容中添加免责声明："本内容部分由AI生成，仅供参考"
- 不将纯AI生成内容用于商标注册、版权登记

## 六、内容侵权（文案/文章/音频/视频）

### 6.1 文案抄袭

- 直接复制他人产品描述、广告语 → 著作权侵权
- "洗稿"（同义替换、结构调整但核心表达相同）→ 仍可能构成侵权
- 使用AI改写他人文章 → 若保留实质性相似，风险仍存在

### 6.2 音乐/音效

| 来源 | 风险 | 替代方案 |
|------|------|---------|
| 流行音乐片段 | 🔴 极高 | Epidemic Sound、Artlist（付费订阅） |
| YouTube免费音乐 | 🟡 中 | 需确认具体Track的许可范围 |
| Freesound.org | 🟢 低 | 注意区分CC0和CC-BY |
| AI生成音乐 | 🟡 中 | Suno/Udio输出权利归属待明确 |

## 七、责任限制条款设计

在应用的用户协议中，建议加入以下条款以降低侵权风险：

```
第X条 知识产权与用户内容

1. 用户使用本应用生成的内容，其知识产权归属由用户与平台另行约定。
2. 用户不得使用本应用生成侵犯第三方知识产权的内容。
3. 如用户上传的内容侵犯第三方权利，用户应承担全部责任。
4. 平台收到侵权通知后，将按照"通知-删除"规则处理。
5. AI生成内容仅供参考，不构成专业建议，平台不对其准确性、合法性作保证。
```
