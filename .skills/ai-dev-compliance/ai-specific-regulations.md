# AI生成内容专项法规深度解析

## 一、中国大陆AI专项法规

### 1.1 《生成式人工智能服务管理暂行办法》（2023.8.15）

#### 适用范围
- 面向中国境内公众提供生成文本、图片、音频、视频等内容的服务
- **不包括**：企业内部使用、未面向公众的研发活动

#### 核心义务

| 义务 | 具体要求 | 违规后果 |
|------|---------|---------|
| **算法备案** | 具有舆论属性或社会动员能力的，需履行备案手续 | 行政处罚、停止服务 |
| **训练数据合法** | 数据来源合法、不侵害知识产权、个人信息需同意 | 民事赔偿+行政处罚 |
| **内容真实准确** | 采取措施防止生成虚假信息 | 视情节处罚 |
| **用户协议** | 明确服务规则、使用限制、投诉渠道 | 行政处罚 |
| **内容标识** | 生成内容可能导致公众混淆的，应显著标识 | 行政处罚 |
| **未成年人保护** | 防止未成年人沉迷、不当使用 | 行政处罚 |

#### 算法备案详解

**哪些需要备案？**
- 具有舆论属性（如新闻、评论、社交）
- 具有社会动员能力（如影响公众意见、组织活动）
- 一般纯工具型应用可能不触发，但建议咨询主管部门

**备案流程**：
1. 通过网信办互联网信息服务算法备案系统填报
2. 提交算法主体信息、算法详细信息、产品及功能信息
3. 备案号需在服务中显著公示

### 1.2 《互联网信息服务深度合成管理规定》（2023.1.10）

#### 适用范围
- 深度合成技术：利用深度学习、虚拟现实等生成或编辑文本、图像、音频、视频、虚拟场景
- 具体包括：人脸生成/替换、语音合成、图像生成、视频生成、数字人

#### 核心义务

| 义务 | 具体要求 |
|------|---------|
| **显著标识** | 可能导致公众混淆或误认的，必须在合理位置/范围内进行显著标识 |
| **不得用于虚假信息** | 不得制作/复制/发布/传播虚假信息 |
| **不得侵犯肖像权** | 使用人脸等生物识别信息编辑，需取得个人单独同意 |
| **数据和技术管理方案** | 深度合成服务提供者需制定安全评估和溯源机制 |
| **安全评估** | 新上线/功能变更需安全评估 |

#### 深度合成标识实务

**必须标识的场景：**
- AI换脸视频/图片
- AI合成语音
- AI生成的"虚拟主播"或数字人
- 可能被误认为真实事件的AI生成图片/视频

**标识方式：**
- 图片/视频：在画面显著位置添加"本内容系AI生成"或"深度合成"标识
- 音频：在开头/结尾添加语音声明
- 文本：在显著位置声明"本文由AI生成"

### 1.3 《互联网信息服务算法推荐管理规定》（2022.3.1）

#### 适用范围
- 应用算法推荐技术（包括生成合成、个性化推送、排序精选、检索过滤、调度决策）
- 提供互联网信息服务

#### 核心义务

| 义务 | 具体要求 |
|------|---------|
| **算法备案** | 同上，具有舆论属性或社会动员能力的需备案 |
| **用户标签透明** | 向用户提供便捷关闭算法推荐服务的选项 |
| **未成年人保护** | 不得向未成年人推送可能引发模仿的危险行为内容；网络游戏服务需落实防沉迷 |
| **劳动者保护** | 调度决策类算法需保障劳动者权益（如外卖骑手） |
| **价格保护** | 不得利用算法实施不合理差别待遇（大数据杀熟） |

### 1.4 《科技伦理审查办法（试行）》（2023.12.1）

#### 适用范围
- 涉及人的科技活动（包括利用个人信息数据进行自动化决策）
- 可能带来伦理风险挑战的科技活动

#### 审查要点
- 是否尊重人格尊严、保护隐私
- 是否公平公正、不歧视
- 风险受益比是否合理
- 是否充分知情同意
- 是否有数据安全保护措施

## 二、国际AI监管动态

### 2.1 欧盟《人工智能法》（AI Act, 2024）

#### 风险分级监管

| 风险等级 | 系统类型 | 要求 |
|---------|---------|------|
| **不可接受风险** | 社会信用评分、实时远程生物识别（执法例外）、操纵潜意识 | **禁止** |
| **高风险** | 关键基础设施、教育、就业、执法、移民、司法 | 需符合性评估、CE标识、风险管理、数据质量、透明度、人类监督 |
| **有限风险** | 聊天机器人、情感识别、生物识别分类 | 需告知用户正在与AI交互 |
| **最小风险** | AI游戏、垃圾邮件过滤 | 无强制要求，鼓励自律 |

#### 通用人工智能模型（GPAI）特殊义务
- 系统性风险模型（训练计算量>10^25 FLOP）：需进行模型评估、对抗测试、事件报告
- 所有GPAI：需技术文档、版权合规（需公开训练数据版权策略）

### 2.2 美国AI监管

- **无联邦统一AI法**，以行业监管+州法+行政令为主
- **拜登行政令（2023.10）**：要求AI开发者向联邦政府报告大型模型训练、安全测试结果
- **州级立法**：加州SB 1047（已否决）、科罗拉多州AI歧视法（2026生效）
- **版权焦点**：AI训练是否构成合理使用（纽约时报诉OpenAI等诉讼进行中）

### 2.3 英国AI监管

- **原则-based 监管**：不单独立法，由现有监管机构（ICO、Ofcom等）按五大原则执行
- 五大原则：安全/稳健性、适当透明/可解释、公平、问责/治理、可争议/救济
- **2024年AI（基础模型）法案**：提案阶段，要求基础模型提供者履行安全义务

## 三、主流AI服务条款关键限制

### 3.1 OpenAI（ChatGPT/GPT-4/DALL-E）

| 限制类型 | 具体规定 |
|---------|---------|
| **禁止用途** | 违法活动、生成恶意软件、欺诈、骚扰、儿童安全、政治竞选（特定）、法律/医疗建议（无免责声明时） |
| **输出权利** | 用户拥有输入和输出的权利，但需遵守服务条款 |
| **竞争监控** | 不得使用服务开发竞争产品或训练竞争模型 |
| **标识要求** | 建议披露AI生成的内容 |

### 3.2 Anthropic（Claude）

| 限制类型 | 具体规定 |
|---------|---------|
| **禁止用途** | 监控/分析个人、自动化决策产生法律/重大财务后果、操纵选举、制造虚假信息、武器研发 |
| **输出权利** | Anthropic不主张对用户输入/输出的权利 |
| **API使用** | 需遵守API服务条款，限制模型蒸馏 |

### 3.3 文心一言/通义千问/智谱（中国大模型）

| 共性限制 | 说明 |
|---------|------|
| 不得生成违法内容 | 政治敏感、色情、暴力、恐怖 |
| 不得生成虚假信息 | 尤其涉及社会事件 |
| 内容安全过滤 | 平台自带审核机制 |
| 备案要求 | 面向公众服务需完成算法备案 |

### 3.4 图像生成服务

| 服务商 | 关键限制 | 商用？ |
|--------|---------|--------|
| Midjourney | 免费计划仅限非商用；不声称版权；不得用于法律诉讼证据 | 付费计划可商用 |
| DALL-E (OpenAI) | 需遵守OpenAI条款；不得生成公众人物、暴力、色情 | 可商用 |
| Stable Diffusion | 开源较宽松；模型文件无额外限制 | 可商用 |
| Adobe Firefly | 训练数据为授权内容，声称商用安全 | 可商用 |
| 文心一格/通义万相 | 需遵守平台条款；不得用于违法场景 | 通常可商用（看具体条款） |

### 3.5 音乐/视频生成

| 服务商 | 关键限制 |
|--------|---------|
| Suno | 付费计划可商用；不转让版权；不得生成模仿特定艺术家的声音 |
| Udio | 类似Suno；Pro/Business计划可商用 |
| Runway/Pika | 付费可商用；不得生成虚假信息、侵权内容 |
| Sora (OpenAI) | 研究阶段，条款严格；水印标识 |

## 四、AI合规实操指南

### 4.1 面向中国公众的AI应用合规检查表

- [ ] **算法备案**：如具有舆论属性或社会动员能力，是否完成备案？
- [ ] **深度合成标识**：AI生成内容是否有"AI生成""深度合成"等显著标识？
- [ ] **训练数据合规**：训练/微调数据是否合法获取？是否含未经授权的版权内容？
- [ ] **用户协议**：是否包含AI生成内容的局限性说明、投诉渠道？
- [ ] **内容审核**：是否建立AI输出的审核机制？是否有有害信息过滤？
- [ ] **虚假信息防范**：是否有机制防止AI被用于生成/传播虚假信息？
- [ ] **未成年人保护**：是否有年龄验证和防沉迷机制？
- [ ] **知识产权声明**：是否声明AI生成内容不构成作品或版权归属待定？

### 4.2 AI输出商用前的检查流程

```
AI生成内容 → 人工审核 → 事实核查 → 版权检查 → 添加标识/免责声明 → 发布
```

### 4.3 免责声明模板

```markdown
关于AI生成内容的声明

1. 本应用部分功能使用人工智能技术生成内容，仅供参考，不构成专业建议。
2. AI生成内容可能存在不准确、不完整或过时之处，请自行核实。
3. AI生成内容的知识产权归属以适用法律和本协议约定为准。
4. 请勿将AI生成内容用于违法、侵权或欺诈目的。
5. 如对AI生成内容有异议，请联系【联系方式】。
```

## 五、典型案例

| 案例 | 背景 | 结果/启示 |
|------|------|----------|
| 菲林律所诉百度（2018） | 百度AI生成报告被诉侵权 | 法院认定AI生成报告不构成作品（无人类独创性），但判决需注明AI生成 |
| 腾讯诉上海盈讯（2020） | 腾讯Dreamwriter AI生成文章被转载 | 法院认定AI文章体现人类独创性安排，构成作品，侵权成立 |
| Thaler诉USCO（2023） | 纯AI生成图像申请版权登记 | 美国法院拒绝：人类作者身份是版权保护前提 |
| 纽约时报诉OpenAI（2023-） | 训练数据侵权 | 进行中，将界定AI训练是否构成合理使用 |
| Midjourney商标案（2023） | 纯AI生成图像申请商标 | USPTO拒绝：需人类作者身份 |
