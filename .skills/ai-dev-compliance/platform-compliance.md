# 平台与分发合规要求

## 一、Apple App Store

### 1.1 隐私营养标签（Privacy Nutrition Labels）

**要求**：2020年12月起，所有App必须在App Store Connect中填写隐私标签，披露收集的数据类型。

**数据类型分类：**
- 联系信息（姓名、邮箱、电话）
- 健康与健身
- 财务信息
- 位置（精确/大致）
- 敏感信息
- 通讯录
- 用户内容
- 浏览历史
- 搜索历史
- 标识符（设备ID）
- 购买记录
- 诊断数据
- 使用数据
- 广告数据
- 其他数据

**合规要点：**
- 标签必须与隐私政策和实际行为一致
- 不一致可能导致拒绝审核或下架
- 即使数据"仅用于App功能"也需披露

### 1.2 App Tracking Transparency（ATT框架）

**要求**：iOS 14.5+，访问设备广告标识符（IDFA）前必须弹窗获得用户许可。

**合规要点：**
- 必须在Info.plist添加`NSUserTrackingUsageDescription`
- 弹窗文案需说明为何需要跟踪
- 用户拒绝后，不得使用其他方式追踪（指纹等）
- 不得因用户拒绝而限制功能（除非该功能本身依赖个性化广告）

### 1.3 支付与内购规则

**核心规则：**
- 数字内容/服务（订阅、游戏币、解锁功能）**必须使用Apple内购**
- 禁止在App内引导用户使用外部支付（"Reader App"等有限例外）
- 2024年起部分市场允许外部支付链接，但需额外审核

**高风险行为：**
- 在App内放置"去官网充值更便宜"等引导文案
- 绕过内购的"兑换码"系统
- 隐藏支付渠道

### 1.4 常见拒审/下架原因

| 原因 | 说明 |
|------|------|
| 2.1 App Completeness | 崩溃、加载失败、功能不完整 |
| 2.3 Accurate Metadata | 描述与实际功能不符、误导性截图 |
| 3.1.1 In-App Purchase | 数字商品未使用内购 |
| 3.2.1 Acceptable | 商业模式不明确、欺诈嫌疑 |
| 4.1 Copycats | 模仿其他App |
| 4.2 Minimum Functionality | 功能过于简单、非App包装网页 |
| 5.1.1 Data Collection and Storage | 收集不必要数据、未提供隐私政策 |
| 5.1.2 Data Use and Sharing | 数据使用超出披露范围 |
| 5.1.3 Health Research | 健康相关功能需额外验证 |

## 二、Google Play

### 2.1 数据安全表单（Data Safety Form）

**要求**：必须在Play Console填写数据安全声明，与隐私政策一致。

**需披露：**
- 收集的数据类型（与Apple类似分类）
- 每种数据是否加密传输
- 数据是否用于特定目的（如广告、分析、欺诈防范）
- 是否共享给第三方
- 是否支持删除请求

### 2.2 权限声明（Permissions Declaration Form）

**高风险权限：**
- `SMS`、`CALL_LOG`：仅特定场景允许（如默认短信/电话应用）
- `LOCATION`：必须有核心功能支撑
- `CAMERA`、`MICROPHONE`：需解释为何必需

**合规要点：**
- 不得申请与应用功能无关的权限
- 权限使用说明必须真实
- 必须在运行时请求权限（Android 6.0+）

### 2.3 家庭政策（Families Policy）

如应用面向儿童或加入"Designed for Families"：
- 不得收集儿童个人信息（13岁以下）
- 不得展示不当广告
- 内容必须适合儿童

### 2.4 常见下架原因

| 原因 | 说明 |
|------|------|
| 欺骗性行为 | 功能与描述不符、模拟系统通知 |
| 不当内容 | 色情、暴力、仇恨言论 |
| 隐私政策缺失 | 未提供或链接无效 |
| 过度索权 | 申请与功能无关的权限 |
| 恶意软件 | 包含病毒、木马、间谍软件 |
| 知识产权侵权 | 使用他人商标/版权内容 |

## 三、微信小程序

### 3.1 基础合规

| 要求 | 说明 |
|------|------|
| **ICP备案** | 涉及web-view、网络请求的小程序需主体有ICP备案 |
| **用户隐私保护指引** | 必须在小程序后台配置《用户隐私保护指引》，与隐私政策一致 |
| **类目资质** | 选择正确类目，部分类目需额外资质（如社交、电商、医疗） |
| **内容安全** | 用户生成内容（UGC）需接入内容安全审核API |
| **HTTPS** | 所有网络请求必须使用HTTPS |

### 3.2 常见违规与处罚

| 违规类型 | 处罚 | 示例 |
|---------|------|------|
| 未配置隐私保护指引 | 限制搜索、分享 | 收集用户信息但未配置 |
| 超范围收集 | 下架 | 收集手机号但无对应功能 |
| 内容违规 | 下架/永久封禁 | 色情、赌博、政治敏感 |
| 诱导分享 | 限制分享 | "分享后才能解锁" |
| 绕开审核 | 永久封禁 | 动态加载违规内容 |
| 侵权 | 下架 | 使用未授权商标/内容 |

### 3.3 用户隐私保护指引配置要点

```
一、信息收集与使用
- 收集的信息类型：【如：昵称、头像、位置】
- 收集目的：【如：展示用户信息、提供附近服务】
- 使用方式：【如：存储于服务器，用于功能实现】

二、信息共享
- 是否共享：是/否
- 共享对象：【如：微信支付、地图服务】
- 共享目的：【如：支付处理、位置展示】

三、信息存储与保护
- 存储地点：【如：中国大陆境内】
- 保护措施：【如：加密存储、访问控制】

四、用户权利
- 访问、更正、删除个人信息的方式
- 注销账号的方式
```

## 四、国内安卓应用商店

### 4.1 工信部APP合规要求

**核心要求（依据《APP用户权益保护测评规范》）：**

1. **隐私政策**：必须提供，且易访问（启动后不超过4次点击）
2. **告知同意**：收集个人信息前需告知目的、方式、范围
3. **最小必要**：收集的信息与功能直接相关
4. **权限对应**：申请的权限有对应功能
5. **账号注销**：必须提供有效的账号注销功能
6. **个性化推荐**：如使用，需提供关闭选项

### 4.2 各商店特色要求

| 商店 | 特色要求 |
|------|---------|
| 华为应用市场 | 严格的权限审查、内容分级、年龄适宜性 |
| 小米应用商店 | 隐私合规检测、绿色应用认证 |
| OPPO/vivo | 类似工信部要求，强调权限最小化 |
| 应用宝 | 内容安全审核严格、社交类需额外资质 |

## 五、Web/浏览器合规

### 5.1 Cookie同意（欧盟/英国）

**要求：**
- 非必要Cookie（分析、广告）必须获得用户明确同意（opt-in）
- 必要Cookie（安全、功能）可无需同意
- 同意记录必须可审计

**合规要点：**
- 不得预勾选非必要Cookie
- 不得因拒绝Cookie而阻止访问（除非该Cookie确为服务所必需）
- 必须提供便捷的撤回同意方式

### 5.2 HTTPS与内容安全

- **HTTPS**：必须使用有效SSL/TLS证书
- **混合内容**：HTTPS页面不得加载HTTP资源（现代浏览器会拦截）
- **CSP（Content Security Policy）**：建议配置以减少XSS风险

### 5.3 无障碍（Accessibility）

| 市场 | 要求 |
|------|------|
| 美国 | ADA Title III可能适用，已有网站被诉案例 |
| 欧盟 | EAA（欧洲无障碍法）2025年6月起适用于多数数字服务 |
| 中国 | 《无障碍环境建设法》2023年，鼓励网站/App无障碍 |

**基础实践：**
- 图片添加alt文本
- 支持键盘导航
- 颜色对比度符合WCAG 2.1 AA标准
- 表单标签关联正确

## 六、云服务与第三方API条款

### 6.1 常见云服务限制

| 服务商 | 免费层限制 | 关键合规点 |
|--------|-----------|-----------|
| **Vercel** | Hobby计划非商用限制；禁止爬虫/挖矿 | 大流量需升级计划 |
| **Netlify** | 100GB/月带宽；禁止加密货币 | 超出配额后服务受限 |
| **Firebase** | Spark计划有配额限制 | 数据存储位置选择 |
| **Supabase** | 免费层有数据库/带宽限制 | 数据存储区域、传输加密 |
| **AWS Free Tier** | 12个月免费 | 超出后按量计费 |
| **Cloudflare** | 免费计划有功能限制 | 不得用于非法内容 |

### 6.2 第三方登录合规

| 平台 | 关键限制 |
|------|---------|
| 微信登录 | 需ICP备案；不得用于非微信生态导流；需配置隐私协议 |
| QQ登录 | 需合规使用，不得超范围获取用户信息 |
| 微博登录 | 需审核应用场景 |
| Google Sign-In | 需符合Google API服务用户数据政策；OAuth同意屏幕需准确 |
| Apple Sign-In | 如支持iOS，Apple要求必须提供Sign in with Apple选项 |

### 6.3 支付接口合规

| 服务商 | 关键合规点 |
|--------|-----------|
| 微信支付 | 需企业资质/个体工商户；需ICP备案；不得用于非法交易 |
| 支付宝 | 类似微信支付 |
| Stripe | 需遵守Stripe服务协议、 prohibited businesses清单 |
| PayPal | 禁止某些高风险的商品/服务类型 |
